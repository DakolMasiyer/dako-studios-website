# Dako Studios — Blog SEO Architecture Template

**Feeds from:** `03_STRATEGY_TEMPLATE.md` + `04_CONTENT_PILLARS_TEMPLATE.md`
**Lives at:** `dako.studio/blog`
**Publishing cadence:** 2x/month minimum

---

## How to Use This Template

1. Fill the 4 cluster definitions using your Project Brief
2. Map the pillar + supporting posts for each cluster
3. Assign keywords and search intent
4. Run the 12-week publishing sequence

The 4-cluster architecture is fixed. Only the topics change.

## 12-Week Publishing Sequence

**Logic:** Low-competition keywords first. Labs/SME content weighted in Weeks 1–6 to match the active sprint; other arms phased in from Week 7.

| Week | Post | Cluster | Keyword | Why this order |
|---|---|---|---|---|
| 1 | `[POST_WEEK_1]` | C1 | `[KW_W1]` | Lowest difficulty. Establishes SME-side foundation. |
| 2 | `[POST_WEEK_2]` | C4 | `[KW_W2]` | Proof content. Establishes brand credibility early. |
| 3 | `[POST_WEEK_3]` | C1 | `[KW_W3]` | Supporting post to Week 1. Builds internal link to pillar. |
| 4 | `[POST_WEEK_4]` | C2 | `[KW_W4]` | Service-workflow pillar. Starts converting SME readers. |
| 5 | `[POST_WEEK_5]` | C4 | `[KW_W5]` | Case study / proof post. Anchor for Cluster 4. |
| 6 | `[POST_WEEK_6]` | C1 | `[KW_W6]` | Supporting post. Links to Weeks 1 + 3. |
| 7 | `[POST_WEEK_7]` | C3 | `[KW_W7]` | Ecosystem pillar. Starts building cultural authority. |
| 8 | `[POST_WEEK_8]` | C2 | `[KW_W8]` | Supporting post to Week 4. Links to service pillar. |
| 9 | `[POST_WEEK_9]` | C3 | `[KW_W9]` | Supporting post. Links to Week 7 pillar. |
| 10 | `[POST_WEEK_10]` | C4 | `[KW_W10]` | Founder update / capstone. Links to all prior Cluster 4 posts. |
| 11 | `[POST_WEEK_11]` | C1 | `[KW_W11]` | Final Cluster 1 supporting post. Completes internal link web. |
| 12 | `[POST_WEEK_12]` | C2 | `[KW_W12]` | Final Cluster 2 post. Full cluster now internally linked. |

## Technical Notes (Framer / Next.js Blog)

- **Dual-date system:** `publishDate` (immediate — for Google indexing) + `socialDate` (reader-facing display — matches social calendar date)
- **Internal linking:** Every supporting post links to its pillar + 1–2 sibling posts in the same cluster
- **Blog-to-social bridge:** Every post ends with a note on which social format it becomes (carousel / thread / single)
- **Academy note:** learn.dako.studio content can be cross-linked from Cluster 1/3 posts where relevant, but Academy SEO/content is otherwise tracked separately from the agency blog
