# Marketing OS — Dako Studios Transfer Plan

**Version:** 1.0 (refined from SyncMaster Marketing OS)
**Source:** SyncMaster Digital Marketing System — restructured for a single-sided, multi-service creative agency
**Purpose:** Build the full Dako Studios marketing OS across all 5 service arms (Labs, Brand, Motion, Film, Academy)

---

## What Changed From the Original Template

The original system was built for a **two-sided marketplace** (a supply side that produces value + a demand side that pays). Dako Studios is a **single-sided service agency** — you sell directly to clients, there's no separate supply pool needing platform access. The biggest structural change: every "Supply persona / Demand persona" split has been replaced with **3 grouped client personas** spanning the 5 arms, and the two parallel outreach tracks have been collapsed into one sales funnel per persona group.

What's preserved because it still works for an agency: the 5-pillar content architecture, the 4-phase content arc, the objection-handling framework, the blog SEO cluster model, and the discipline of never inventing proof points.

## How to Use This System

### Step 1 — Fill the Project Brief (Day 1)
Open `01_PROJECT_BRIEF.md`. Answer all 9 questions. This becomes the seed document for everything else. Do not skip this step.

### Step 2 — Define Personas (Day 1–2)
Open `02_PERSONA_TEMPLATE.md`. Use your Project Brief answers to fill the 3 grouped client personas: SME/Business Owner, Film/Entertainment Client, Aspiring Creative.

### Step 3 — Write the Strategy (Day 2–3)
Open `03_STRATEGY_TEMPLATE.md`. Replace every `[PLACEHOLDER]` using your brief and persona docs. This is your master strategy document — everything else is derived from it.

### Step 4 — Build the Content System (Day 3–5)
Fill `04_CONTENT_PILLARS_TEMPLATE.md` → then `05_CONTENT_CALENDAR_TEMPLATE.md` → then `06_COPY_BANK_TEMPLATE.md` → then `08_BLOG_SEO_TEMPLATE.md`. Work in this order — pillars inform the calendar, the calendar informs copy, copy informs SEO topics.

### Step 5 — Build the Outreach System (Day 5–7)
Fill `07_OUTREACH_TEMPLATE.md`. These sequences turn your strategy into actual conversations with prospective clients, weighted toward Labs first.

### Step 6 — Arm the Content with Objection Handling (Day 7)
Fill `09_OBJECTION_HANDLING_TEMPLATE.md`. Use the objections to stress-test every content piece and outreach message before it goes live.

### Step 7 — Build the Tech Stack (Day 7–14)
- Airtable ops base (pipeline tracker — leads, proposals, clients)
- Tally or web form (lead intake per arm)
- Resend or similar email sequences (from outreach template)
- Framer sites (Labs delivery tool — client-editable CMS, fast turnaround)
- learn.dako.studio (Academy — already live, separate funnel)
- Portfolio/case study deck (credibility asset, fork pattern from any existing deck)

### Step 8 — Launch Sequence (Day 14–21)
1. Deploy the portfolio/case study deck — instant credibility asset
2. Publish first 5 content pieces (proof + education mix), Labs-weighted
3. Open lead capture (web form / DM) for Labs
4. Begin cold outreach for the Labs persona (SME/Business Owner)
5. Document first proof point publicly as soon as it exists
6. Phase in Brand/Motion/Film/Academy outreach as Labs pipeline stabilizes (Month 2–3)

## What Changes vs. What Stays the Same

### Changes for every new arm or campaign
- Persona specifics (pain language, proof points, channel mix) per arm
- Proposal pricing, timelines, and packages
- Portfolio pieces and case study details
- Platform-specific handle names

### Never changes (the reusable OS)
- 5-pillar content architecture
- 4-phase content arc (Prove → Build Trust → Demonstrate Scale → Compound)
- Single-funnel outreach structure (Awareness → Lead → Proposal → Client → Referral)
- Objection handling framework
- Blog SEO cluster architecture
- Layer map (L1 → L5)
- Voice principles (direct, specific, building in public, confident)

**Fill `01_PROJECT_BRIEF.md` before touching any other file.**
Every template has placeholders that map directly to the 9 questions in the brief. If the brief is incomplete, the templates will be hollow. If the brief is sharp, the rest fills fast.
